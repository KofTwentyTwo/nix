# Pi Coding Agent Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install pi (Mario Zechner's minimal terminal coding harness) as a daily-driver coding agent, fully nix-managed, with cloud-default routing (`claude-opus-4-7` via Claude Pro/Max subscription) and local Ollama fallback (Qwen 3 Coder 30B, Qwen 2.5 Coder 7B, Llama 3.3 70B via MLX backend).

**Architecture:** New `home/pi/` Home Manager module writes `~/.pi/agent/{models.json,SYSTEM.md,AGENTS.md}` via `home.file`. Activation script pulls Ollama models idempotently. API keys via `!op read` 1Password shell-command auth — no secrets on disk. The pi npm package and ollama-app cask are already installed; this plan adds only the configuration layer.

**Tech Stack:** Nix (flakes), nix-darwin, Home Manager, Ollama (with MLX backend), `@mariozechner/pi-coding-agent`, 1Password CLI (`op`).

**Spec:** `docs/superpowers/specs/2026-05-20-pi-setup-design.md`

## Pre-flight: assumptions

This plan assumes:
- pi binary is already on PATH (verified Task 1)
- ollama CLI is on PATH and the daemon is running (verified Task 1)
- 1Password CLI (`op`) is available and signed in (verified Task 1)
- `~/.ai/3-rules.md` and related files exist (referenced by AGENTS.md)
- Claude Pro/Max subscription is active (default model uses it)
- Apple Silicon Mac with 32 GB+ unified memory (MLX backend requirement)

### Task 5: Create `home/pi/default.nix`

**Files:**
- Create: `home/pi/default.nix`

- [ ] **Step 1: Write default.nix**

Create `~/.config/nix/home/pi/default.nix`:

```nix
# Pi Coding Agent Configuration Module
# =====================================
# Configures Mario Zechner's pi coding agent (@mariozechner/pi-coding-agent),
# already installed via home/npm-globals/default.nix.
#
# Files managed:
#   - ~/.pi/agent/models.json: Providers + models (rendered from models.nix)
#   - ~/.pi/agent/SYSTEM.md:   Pi-specific behaviors
#   - ~/.pi/agent/AGENTS.md:   Pointer to ~/.ai/* chain
#
# Activation: pulls Ollama models idempotently (skips if present).
# First switch may take ~10 minutes for the ~70 GB download.

{ config, pkgs, lib, ... }:

let
  modelsData = import ./models.nix { inherit config lib; };
in
{
  home.file = {
    ".pi/agent/SYSTEM.md".source = ./SYSTEM.md;
    ".pi/agent/AGENTS.md".source = ./AGENTS.md;
    ".pi/agent/models.json".text = builtins.toJSON modelsData;
  };

  # Pull Ollama models idempotently. First switch may take ~10 minutes for the
  # full ~70 GB download; subsequent switches skip existing models in seconds.
  # Non-fatal: continues activation even if a pull fails (offline, etc).
  home.activation.pullPiModels = lib.hm.dag.entryAfter [ "writeBoundary" ] ''
    if command -v ollama >/dev/null 2>&1; then
      for m in qwen3-coder:30b-a3b-q6_k qwen2.5-coder:7b llama3.3:70b-instruct-q4_K_M; do
        if ! ollama list 2>/dev/null | grep -q "''${m}"; then
          echo "pi: pulling Ollama model ''${m} (may take several minutes)"
          $DRY_RUN_CMD ollama pull "''${m}" || \
            echo "pi: WARN pull of ''${m} failed; continuing" >&2
        fi
      done
    else
      echo "pi: WARN ollama not on PATH; skipping model pulls" >&2
    fi
  '';
}
```

No commit yet — the module is not imported, so `darwin-rebuild` won't touch it.
