## CRITICAL (fix immediately)

### [ ] 1. AWS Credentials Exposed in Nix Store

**File:** `home/aws/default.nix` (line 52) and `home/aws/config/credentials`

**Problem:** The AWS credentials file contains real access keys (profile `dmd-brands-audit-cli` with key `AKIA5GJIXNMQOZXRMXVE`). `home.file` copies it into the Nix store which is world-readable (`0444`). The activation script's `chmod 600` silently fails because the Nix store is immutable. Any local user can read these credentials.

**Fix:** Stop using `home.file` for credentials. Options:
1. Use `home.activation` to copy the file directly (not via Nix store) with correct permissions
2. Use `sops-nix` or `agenix` for secrets management
3. Load credentials via 1Password CLI (`op run`) instead of storing on disk
4. At minimum, use `home.file."...".onChange` with a copy-and-chmod script

### [ ] 2. `git gc` Overridden by Commitizen Alias

**File:** `flake.nix` line 306

**Problem:**
```nix
alias.gc = "!cz";  # Make git gc run commitizen
```
This overrides `git gc` (garbage collection), a critical built-in git command. Running `git gc` launches commitizen instead. Repos will bloat over time and automated tools that call `git gc` will break.

**Fix:** Remove this line entirely. The zsh shell alias `gc = "git cz c"` in `home/zsh/default.nix` already provides the commitizen shortcut.

### [ ] 3. `nix.enable = false` Missing on Grogu

**File:** `flake.nix` lines 367-380 (Grogu darwinConfiguration block)

**Problem:** Darth (line 356) and Renova (line 387) set `nix.enable = false;` but Grogu does not. This means nix-darwin will try to manage the Nix daemon on Grogu, conflicting with any existing Nix installation.

**Fix:** Add `nix.enable = false;` to the Grogu configuration block, matching Darth and Renova.

### [ ] 11. settings.json Merge May Wipe Manual Plugins

**File:** `home/claude/default.nix` line 424

```bash
jq '. * $prefs[0]' "$user_settings"
```

**Problem:** The `*` operator in jq replaces entire nested objects. If `enabledPlugins` is in `$prefs[0]`, it completely replaces the on-disk `enabledPlugins`, wiping any manually-added plugins. This is now mitigated since we synced the Nix list to match reality, but future manual additions will be lost on rebuild.

**Fix:** Change to a deep merge that preserves keys not in the Nix config:
```bash
jq --slurpfile prefs "$userPrefsJson" '
  .enabledPlugins = (.enabledPlugins // {}) * $prefs[0].enabledPlugins
  | . * ($prefs[0] | del(.enabledPlugins))
' "$user_settings"
```
Or accept that the Nix config is the single source of truth and always add new plugins there.
